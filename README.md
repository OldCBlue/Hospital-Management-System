# 医療情報管理システム (Hospital Information System)

患者管理・診療・薬剤処方・決済機能を統合した、医療従事者向けのフルスタックWeb管理システムです。

## 🏥 プロジェクト概要
本プロジェクトは、医療現場における情報の正確な管理と、業務フローの効率化を目的として開発しました。
特に、大規模なデータ処理におけるセキュリティリスク（認証・認可）の低減と、保守性の高いアーキテクチャの構築に注力しました。

## 🛠 使用技術 (Tech Stack)
- **Backend**: Java 17 / Spring Boot / Spring Security / MyBatis
- **Frontend**: Vue  / Vite / Element Plus / Axios
- **Infrastructure & Middleware**: Redis (キャッシュ・認証管理) / MySQL / Docker

## ✨ 技術的な工夫と改善点
### 🔐 認証・認可の高度化 (JWT + Redis)
- **課題**: 従来のセッション管理では、将来的なサーバーの水平分散（スケーリング）時にセッション共有の整合性が取れなくなるリスクがありました。また、認証処理のオーバーヘッドによるセキュリティリスクも懸念事項でした。
- **解決**: **JWT (JSON Web Token)** を導入し、**Redis** でトークンの有効性を管理するステートレスな認証基盤を構築。セキュアで高速なログイン状態の管理を実現しました。
- **実装の詳細**：
      ログイン成功時、Redisにトークンを保存し、有効期限（TTL）を動的に管理することで、ログアウト処理や強制ログオフを確実に制御可能にしました。
      Redisをキャッシュ層として利用することで、DBへのクエリ回数を減らし、高速な認可チェックを実現しました。

### 🧵 スレッド安全性の確保 (ThreadLocalによるコンテキスト管理)
- **課題**: SpringBootのマルチスレッド環境において、複数のリクエストが同時に発生した際、ユーザー情報が他のスレッドに混入する（データ競合）リスクがありました
- **解決**: **ThreadLocal** を活用してユーザーコンテキストをカプセル化し、リクエストごとのデータ独立性を完全に担保しました。
- **実装の詳細**：認証フィルター通過後、ユーザーID等のコンテキスト情報をThreadLocalに格納し、サービス層やDAO層から安全にアクセスできる仕組みを構築しました。
    リクエスト終了時には必ず remove() を実行するよう例外処理を徹底し、スレッドプール再利用時のメモリリークや意図しない情報の残存を防止しました。
### 🏗 アーキテクチャの刷新 (DTO / VO)
- **課題**:エンティティ（DB構造）をそのままフロントエンドに返していたため、不要な内部データの露出や、APIの柔軟性不足が保守性を低下させていました。
- **解決**: **DTO (Data Transfer Object)** と **VO (View Object)** を厳格に分離。APIレスポンスを最適化し、拡張性の高いクリーンな設計へと刷新しました。
- **実装の詳細**：DTO: フロントエンドからの入力を受け取り、業務ロジックに必要な最小限のデータとして定義しました。
  VO: 表示に必要な項目のみをカプセル化し、APIレスポンスのサイズ軽量化とセキュリティの向上を図りました。
  この分離により、バックエンドのDB設計変更がフロントエンドに直接影響を与えない疎結合な設計を実現しました。
